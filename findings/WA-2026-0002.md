# Falta de rate limiting en login

**ID:** WA-2026-0002
**Severidad:** Medium
**OWASP Top 10:** A07 — Identification and Authentication Failures
**OWASP WSTG:** WSTG-AUTH-03
**ASVS:** 2.1.1, 2.2.3
**Estado:** Open
**Fecha:** 2026-07-07
**Auditor:** Security Team

---

## Descripción

El endpoint `/api/v1/auth/login` en IAM no implementa rate limiting ni bloqueo de cuenta tras intentos fallidos. No hay límite en la frecuencia de requests de autenticación.

## Impacto

Un atacante puede realizar ataques de fuerza bruta y diccionario contra cuentas de usuario sin restricción.

## Probabilidad

Alta — No hay ninguna defensa contra brute force.

## Condición

- Endpoint público sin autenticación
- Sin rate limiting, captcha, o bloqueo de cuenta

## Reproducción

```bash
# Intentos ilimitados de login en ráfaga
for i in $(seq 1 1000); do
  curl -X POST http://localhost:8100/api/v1/auth/login \
    -d "username=admin@verp.com&password=password$i"
done
```

## Evidencia

`IAM/src/iam/domains/auth/router.py:15-26`:
```python
@router.post("/login", response_model=schemas.TokenPair)
async def login(
    form_data: Annotated[OAuth2PasswordRequestForm, Depends()],
    session: Annotated[AsyncSession, Depends(get_db_session)],
) -> schemas.TokenPair:
    return await service.login(session, email=form_data.username, password=form_data.password)
```
No hay decoradores de rate limiting ni lógica de bloqueo.

## Recomendación

- Implementar rate limiting (slowapi o middleware propio)
- Bloquear cuenta tras N intentos fallidos (ej: 5)
- Añadir soporte para Captcha/CAS
- Logging de intentos fallidos para detección temprana
