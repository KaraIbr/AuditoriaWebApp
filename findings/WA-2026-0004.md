# Security headers ausentes

**ID:** WA-2026-0004
**Severidad:** Low
**OWASP Top 10:** A05 — Security Misconfiguration
**OWASP WSTG:** WSTG-CONF-04
**ASVS:** 12.1.1, 12.5.1
**Estado:** Open
**Fecha:** 2026-07-07
**Auditor:** Security Team

---

## Descripción

Las APIs de IAM y CRM no configuran security headers HTTP: Content-Security-Policy, X-Frame-Options, X-Content-Type-Options, Strict-Transport-Security, ni Referrer-Policy.

## Impacto

- Sin CSP: mayor exposición a XSS (exfiltración de datos)
- Sin X-Frame-Options: riesgo de clickjacking
- Sin HSTS: riesgo de downgrade attacks (HTTP → HTTPS)
- Sin X-Content-Type-Options: riesgo de MIME sniffing

## Probabilidad

Baja — React y FastAPI tienen protecciones incorporadas parciales.

## Condición

- Frontend servido por Vite sin CSP configurado
- APIs FastAPI sin middleware de seguridad

## Evidencia

Análisis de código en `IAM/src/iam/core/config.py` y routers — no se encontró configuración de security headers. No hay middleware de seguridad en ninguna de las aplicaciones FastAPI.

## Recomendación

- Agregar middleware que inyecte security headers en todas las respuestas
- Configurar CSP restrictivo para el frontend
- Habilitar HSTS en producción con `includeSubDomains`

## Código sugerido

```python
from fastapi import FastAPI
from starlette.middleware.base import BaseHTTPMiddleware

async def security_headers_middleware(request, call_next):
    response = await call_next(request)
    response.headers["X-Content-Type-Options"] = "nosniff"
    response.headers["X-Frame-Options"] = "DENY"
    response.headers["Strict-Transport-Security"] = "max-age=31536000; includeSubDomains"
    response.headers["Referrer-Policy"] = "strict-origin-when-cross-origin"
    return response

app.add_middleware(BaseHTTPMiddleware, dispatch=security_headers_middleware)
```
