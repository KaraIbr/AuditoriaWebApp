# Dev mode bypass de autenticación

**ID:** WA-2026-0005
**Severidad:** Low
**OWASP Top 10:** A05 — Security Misconfiguration
**OWASP WSTG:** WSTG-AUTH-04
**ASVS:** 2.1.1
**Estado:** Open
**Fecha:** 2026-07-07
**Auditor:** Security Team

---

## Descripción

En modo development (`environment=development`), la dependencia `get_current_user` retorna automáticamente el usuario con ID 1 si no se provee token JWT. Esto bypassa la autenticación completa.

## Impacto

Si un servidor en modo development está expuesto (inadvertidamente), cualquier request sin token obtiene acceso como admin.

## Probabilidad

Baja — Solo aplica en modo development.

## Condición

- `IAM_ENVIRONMENT=development` (default)
- Sin token JWT en el request
- Usuario con ID 1 existe en la DB

## Evidencia

`IAM/src/iam/api/dependencies.py:42-45`:
```python
if token is None and settings.is_development:
    user = await get_active_user_by_id(session, 1)
    if user is not None:
        return user
```

## Recomendación

- Deshabilitar este bypass en producción (documentado como seguro)
- Asegurar que servidores development no sean accesibles desde redes externas
- Considerar remover esta funcionalidad a favor de un seed script dedicado
