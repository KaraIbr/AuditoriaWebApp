# Logout no invalida tokens server-side

**ID:** WA-2026-0003
**Severidad:** Low
**OWASP Top 10:** A07 — Identification and Authentication Failures
**OWASP WSTG:** WSTG-SESS-06
**ASVS:** 3.1.1
**Estado:** Open
**Fecha:** 2026-07-07
**Auditor:** Security Team

---

## Descripción

El endpoint `/api/v1/auth/logout` en IAM no invalida el token JWT server-side. Es un no-op que solo responde 204 sin realizar ninguna acción de revocación.

## Impacto

Un token JWT robado sigue siendo válido hasta su expiración natural (30 min para access, 7 días para refresh) incluso después de que el usuario legítimo cierre sesión.

## Probabilidad

Media — Requiere que el token haya sido comprometido previamente.

## Condición

- El usuario hace logout
- El token sigue siendo válido
- Un atacante con el token puede seguir usándolo

## Evidencia

`IAM/src/iam/domains/auth/router.py:39-43`:
```python
@router.post("/logout", status_code=status.HTTP_204_NO_CONTENT)
async def logout(_: CurrentUser) -> Response:
    """Accept logout requests; token revocation is a future persisted concern."""
    return Response(status_code=status.HTTP_204_NO_CONTENT)
```

El comentario confirma que es una funcionalidad diferida.

## Recomendación

- Implementar una blacklist de tokens (Redis o DB) para invalidar tokens post-logout
- Usar JWT con jti (JWT ID) para identificar tokens individualmente
- Mantener TTL en la blacklist igual al tiempo de expiración del token
